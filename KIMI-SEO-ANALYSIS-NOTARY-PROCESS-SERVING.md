## HOW TO CRAWL

**For each URL below, use your browsing/fetch tool to open the page.** Then:

### Visual Inspection (What Does the User See?)
**→ Use your browsing tool to load the page. Read the full HTML output.**
- Evaluate the hero section / above-the-fold content
- Check if CTAs are visible and compelling
- Assess visual hierarchy, layout, whitespace, readability
- Note any broken images, layout shifts, or visual bugs
- Check mobile responsiveness (does it look good at 375px, 768px, 1024px, 1440px?)
- Evaluate color contrast and accessibility (WCAG 2.1 AA compliance)
- Note the overall professional impression (would a lawyer trust this site?)

### Code-Level Inspection (What Does Google See?)
**→ From the same fetched HTML, inspect the raw source code. Look at the `<head>` and `<body>` tags.**
- View page source / inspect the DOM
- Check `<title>` tag — is it unique, keyword-rich, under 60 characters?
- Check `<meta name="description">` — is it compelling, under 160 characters, includes CTA?
- Check `<meta name="robots">` — is it indexed/followed?
- Check Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`)
- Check Twitter Card tags
- Check canonical URL (`<link rel="canonical">`)
- Check all heading hierarchy (`<h1>` through `<h6>`) — is there exactly one `<h1>`? Are headings semantic and keyword-optimized?
- Check for structured data / JSON-LD schema markup — what types are present? Are they valid?
- Check internal links — how many? Where do they point? Are there orphan pages?
- Check external links — are they relevant? Do they use `rel="noopener"` / `rel="nofollow"` where needed?
- Check image `alt` text — is every image described properly?
- Check page load performance — are images optimized? Is JavaScript blocking rendering?
- Check for broken links (404s)
- Check `<html lang="en">` attribute
- Check favicon and touch icons

## WHAT TO ANALYZE ON EVERY PAGE

For each page you visit, evaluate and document ALL of the following:

### 1. Content Quality
- Is the content original, accurate, and up to date?
- Is the word count sufficient for the topic? (Aim: 800–2,500 words for service pages, 1,500–4,000 for guides)
- Is the content written for the target audience (60% consumers, 40% attorneys/legal professionals)?
- Are there any factual errors, outdated information, or placeholder text?
- Is the tone professional but approachable?
- Are there spelling or grammar issues?
- Is the content substantive or thin/filler?

### 2. SEO — On-Page
- `<title>` tag: unique, keyword-rich, under 60 chars?
- `<meta description>`: compelling, under 160 chars, includes CTA?
- `<h1>`: exactly one per page, includes primary keyword?
- Heading hierarchy: logical `<h2>` → `<h3>` → `<h4>` flow?
- Keyword density: is the primary keyword used naturally 3–5 times?
- Are LSI/related keywords present throughout the content?
- Internal links: at least 3–5 links to other site pages?
- External links: any authoritative outbound links (Oklahoma Statutes, court websites, etc.)?
- URL slug: clean, keyword-rich, no unnecessary words?
- Image alt text: descriptive, includes keywords where natural?
- Canonical tag: present and pointing to the correct URL?

### 3. SEO — Technical
- Page load speed (is it fast or bloated?)
- Mobile responsive (check at 375px, 768px widths)
- Core Web Vitals concerns (layout shift, large images, render-blocking JS)
- Is the page indexed? (check `robots` meta, `noindex` directives)
- Proper Open Graph and Twitter Card tags?
- JSON-LD structured data — what schemas are present? Are they valid? What's missing?
- Breadcrumb navigation present?
- Sitemap inclusion — is this page in the XML sitemap?

### 4. Schema Markup Audit
For every page, document:
- What JSON-LD schemas are currently implemented
- Whether they pass Google Rich Results Test validation
- What schemas SHOULD be added. Consider:
  - `LocalBusiness` (for service pages)
  - `LegalService` (for process serving pages)
  - `NotaryService` or `Service` (for notary pages)
  - `FAQPage` (for any page with Q&A content)
  - `BreadcrumbList`
  - `Organization`
  - `WebPage` / `WebSite`
  - `HowTo` (for step-by-step content)
  - `Article` / `BlogPosting` (for blog content)
  - `Offer` (for pricing pages)
  - `AggregateRating` / `Review` (if reviews are shown)
  - `AreaServed` / `GeoShape` (for service area pages)

### 5. Conversion & UX
- Is there a clear CTA above the fold?
- Is the phone number clickable (`tel:` link)?
- Are contact forms easy to find and use?
- Is pricing transparent or hidden?
- Are trust signals visible (license numbers, bonding info, years in business, reviews)?
- Is there social proof (testimonials, review counts, star ratings)?
- Is the navigation intuitive? Can users find what they need in 2 clicks?
- Are there any dead ends (pages with no next action)?

### 6. Accessibility (WCAG 2.1 AA)
- Sufficient color contrast ratios?
- All images have meaningful alt text?
- Keyboard navigable?
- ARIA labels where needed?
- Skip navigation links?
- Focus indicators visible?

### 7. Internal Linking & Site Architecture
- How many internal links does this page have?
- Are important related pages linked?
- Is this page linked FROM other important pages?
- Does the breadcrumb trail make sense?
- Are there orphan pages (not linked from anywhere)?
