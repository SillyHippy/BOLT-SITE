## 2024-05-24 - Focus visible styling on custom radio buttons
**Learning:** Custom interactive elements (like `button`s styled as radio groups) often lack default focus indicators when using Tailwind, particularly when generic `outline-none` classes are present. Keyboard accessibility requires explicit `:focus-visible` styling to allow users to navigate these elements via Tab.
**Action:** When creating or modifying custom interactive elements, always explicitly define keyboard focus visibility using Tailwind classes like `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-navy focus-visible:ring-offset-2`.
